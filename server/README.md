# Visual Physics — сервер (заявки + Telegram + ИИ-чат)

Лёгкий backend **без единой внешней npm-зависимости**. Работает на чистом Node.js
(встроенные `node:http`, `node:sqlite`, `fetch`). Нужен только **Node.js 22.5+**.

Что умеет:
- 🗄️ Сохраняет заявки из формы в базу данных **SQLite** (`server/data/leads.db`)
- 📲 Шлёт уведомление о каждой заявке в **Telegram** (кто оставил заявку)
- 🤖 Отвечает в **чат-боте** на сайте (FAQ без ключа, OpenAI или Claude)
- 👀 Показывает список заявок на странице **`/admin`**

---

## 1. Быстрый старт

```bash
cd server
copy .env.example .env       # Windows (или: cp .env.example .env)
npm start
```

Откройте **http://localhost:3000** — это весь сайт. Заявки уже сохраняются в базу,
чат-бот работает в режиме FAQ. Telegram и ИИ подключаются ниже (по желанию).

> Зависимости ставить не нужно — `npm install` не требуется.

---

## 2. Подключить Telegram (уведомления о заявках)

1. В Telegram откройте **@BotFather** → команда `/newbot` → задайте имя → получите **токен**.
2. Впишите токен в `server/.env`:
   ```
   TELEGRAM_BOT_TOKEN=123456:AA....
   ```
3. Напишите своему боту в Telegram любое сообщение (например, «привет»).
   Если уведомления нужны в группу — добавьте бота в группу и напишите там.
4. Узнайте `chat_id`:
   ```bash
   npm run chat-id
   ```
   Скопируйте показанный `chat_id` в `.env` (можно несколько через запятую):
   ```
   TELEGRAM_CHAT_ID=987654321
   ```
5. Перезапустите сервер (`npm start`). Теперь каждая заявка приходит в Telegram.

---

## 3. Админка заявок

1. В `.env` задайте секретный пароль:
   ```
   ADMIN_TOKEN=придумайте-длинный-пароль
   ```
2. Откройте **http://localhost:3000/admin**, введите этот пароль — увидите таблицу заявок.

---

## 4. Включить полноценный ИИ-чат (необязательно)

По умолчанию чат-бот отвечает по базе знаний о проекте (бесплатно, без ключа).
Чтобы сделать его полноценным ИИ-ассистентом, рекомендуемый вариант — OpenAI:

1. Создайте API-ключ в настройках API-проекта OpenAI.
2. В `server/.env` добавьте:
   ```ini
   AI_PROVIDER=openai
   OPENAI_API_KEY=sk-...
   # Необязательно: OPENAI_MODEL=gpt-4.1-mini
   ```
3. Перезапустите сервер. Ключ остаётся только на сервере: браузер его не получает.

Claude тоже поддерживается для обратной совместимости:

```ini
AI_PROVIDER=anthropic
ANTHROPIC_API_KEY=sk-ant-...
# Необязательно: CHAT_MODEL=claude-haiku-4-5-20251001
```

Если ключ отсутствует или ИИ-провайдер временно недоступен, бот автоматически отвечает по локальной базе знаний и сайт не ломается.

---

## 5. Проверка состояния

`http://localhost:3000/api/health` → покажет, подключены ли Telegram и ИИ, активный провайдер (`ai_provider`) и количество заявок в базе.

---

## API

| Метод | Путь          | Описание                                            |
|-------|---------------|-----------------------------------------------------|
| POST  | `/api/lead`   | Принять заявку → БД + Telegram. Тело: `{name, phone, school?, role?, message?}` |
| POST  | `/api/chat`   | Чат-бот. Тело: `{message, history?}`                |
| GET   | `/api/leads`  | Список заявок (нужен `?token=ADMIN_TOKEN`)          |
| GET   | `/api/health` | Состояние сервиса                                   |

---

## Деплой

Подойдёт любой хостинг с Node.js 22.5+ (Render, Railway, Fly.io, VPS).
Команда запуска — `npm start`, переменные окружения — из раздела `.env`.
Папку `server/data/` (база) и `.env` (секреты) в git коммитить не нужно — они уже в `.gitignore`.
